User Guide

User Guide > Document Repository: GitHub

This guide will help you set up GitHub, create and manage a code repository, and integrate it with IntegriLAB to efficiently track and update your projects.

Step 1: Create a GitHub Account
  • Go to the GitHub website and click Sign Up.
  • Fill in your details and follow the instructions to set up your GitHub account.
Step 2: Create New Project and Connect Locally
  • Log in to your GitHub account and create Repository.
  • Clone your GitHub repository to your local machine:
    git clone https://github.com/<username>/< repository_name>.git
  • Navigate to the cloned repository directory:
    cd <repository_name>
  • Add or modify your project files in the local repository.
Step 3: Commit Changes
  • Stage your changes:
    git add .
  • Commit the changes with a message:
    git commit -m "Initial commit"
  • Push the changes to GitHub:
    git push origin main
Step 4: Connect IntegriLAB with GitHub
  • Log in to your IntegriLAB account and select Code option.
  • Navigate to the Setup Connection section in the dashboard.
  • Enter your username, password/token when prompted.
  • Once authenticated, your DataLad repository will be linked to IntegriLAB.
Step 5: Add the GitHub Repository to IntegriLAB
  • In the Code section, choose the Add New Repository option.
  • Associate the GitHub code with the corresponding project in IntegriLAB by using repository name.
Step 6: Update and Commit Changes
  • When updates are made to the code repository, commit and push the changes locally:
    git add .
    git commit -m "Updated code"
    git push origin main
  • To save updates:
    1. Go to the Code -> Home repository and click on Commit Changes in IntegriLAB.
    2. Select the linked project and the related code repository to synchronize the latest version with IntegriLAB.
Repeat this step whenever significant updates are made to ensure all changes are documented.
